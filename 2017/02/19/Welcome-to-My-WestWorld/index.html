版本回退

head指针

head指向的版本就是当前的版本，可以使用git reset --hard commit_id来返回指定的版本

git reset --hard HEAD^返回上个版本，依此类推。版本跨越过多就用
git reset --hard HEAD~100 ，表示返回之前第100个版本

git log 和 git reflog

git log 查看提交历史以便找到所需回退版本的commit_id

git reflog 查看命令历史以便撤销回退操作，用head指向回退前版本的（指令）command_id

git log进入后一直是在log中搜索模式，“q”键可一键退出git log

撤销修改

  • 不小心弄乱了工作区的某个文件内容，想丢弃工作区的修改可用git checkout -- file 注意：中间的–很重要，若没有则是切换分支

  • 不小心弄乱了还添加到暂存区了（执行过add），则用git reset HEAD file(如readme.txt)即可使其回到工作区，再用上一步骤撤销。

删除文件

在工作区域删除文件后

  • 可以用git rm file删除版本库里的文件，再commit即可；

  • 删错了想恢复则用git checkout -- file即可；这里的命令其实是用版本库里的文件替代工作区的，无论删除还是修改都可以一键还原

分支操作

git鼓励大量使用分支

查看分支：git branch

创建分支：git branch <name>

切换分支：git checkout <name>

创建+切换分支：git checkout -b <name>

合并某分支到当前分支：git merge <name>

删除本地分支：git branch -d <name>

问题描述：删除远程分支。

问题解决：

git branch -d <name> 只可以用来删除本地分支，且删除本地分支前应先删除
远程分支否则无法track 远程分支导致无法删除。
